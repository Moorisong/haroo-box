# 📱 전투 다마고치 - UI/UX 및 페이지 상세 명세

본 문서는 전투 다마고치의 각 라우트별 화면 구성 요소와 핵심 사용자 경험(UX) 요구사항을 정의합니다.

---

## 1. 페이지별 구성 요소 (Routes)

### 1.1 메인 홈 (`/tama`)
*   **알림 영역**: 위급 상태 알림, 스탯 경고 등 다마고치의 현재 요구사항 출력.
*   **캐릭터 디스플레이**: 다마고치 픽셀 아트 및 현재 상태 표정 렌더링. 이름, 대표 칭호, 나이, 가문명 정보 노출.
*   **스탯 패널**: 행복도, 배고픔, 청결도, 용기 수치 게이지바 형식으로 시각화.
*   **상호작용 툴바**:
    *   `쓰다듬기`, `박수치기`
    *   `책 읽어주기`, `간식 주기`
    *   `산책`, `목욕`
    *   `장애물 달리기`, `전투` (남은 사용 횟수 병기)
*   **인벤토리**: 산책 등으로 획득한 장착 가능 장식(꽃) 목록 표시 및 선택 토글.

### 1.2 전투 페이지 (`/tama/battle`)
*   **메인**: 오늘 남은 전투 기회 하트 아이콘 표시, [대결 신청], [대결 요청 확인] 메뉴 및 전투 랭킹 링크.
*   **신청 화면**: 나의 다마고치와 상대 다마고치의 칭호, 가문, 전적 정보 비교. 스킬 6종 선택 라디오 버튼 및 [대결 신청하기] 버튼 제공.
*   **전투 대기실**: 10분간의 실시간 카운트다운 타이머 연동 및 시간에 따른 병맛 텍스트 로그 자동 갱신 영역.
*   **결과**: 승리/패배 그래픽스, 획득/차감 스탯 수치 애니메이션, [결과 공유하기] 소셜 링크 연동.

### 1.3 광장 페이지 (`/tama/plaza`)
*   **상단**: 이름 정확도 기준 검색 필드.
*   **접속/활동 유저 리스트**: 실시간 접속자 최대 5인, 최근 활동 유저 최대 5인의 프로필 카드 형태(전투, 결혼 신청 바로가기 포함) 출력.
*   **오늘의 외치기**: 1일 1회 소통 한줄평 게시판 및 실시간 댓글, 좋아요 카운터.

### 1.4 랭킹 페이지 (`/tama/rank`)
*   **정렬 탭**: 최강 전사(전투 승리), 겁쟁이(도망), 행복왕(행복 100 달성), 칭호왕(칭호 수).
*   **랭크 리스트**: 1~3위 특별 금/은/동 메달 표시 및 순위별 가문명, 칭호, 정렬 스탯 값 테이블 뷰. 공동 순위 처리 지원.

### 1.5 칭호 도감 (`/tama/title`)
*   **탭 분리**: 개체 칭호 / 가문 칭호.
*   **카드 리스트**: 획득 완료(컬러 활성화, 체크마크) 및 미획득(자물쇠 잠금, 획득 요구 스탯 및 진행률 달성도 바 제공).

### 1.6 가문 페이지 (`/tama/family`)
*   **가문 스펙**: 가문명, 시조 정보, 창립일, 현재 세대, 가문 연령.
*   **대표 가문 칭호**: 장착 가문 칭호 출력.
*   **계승 컨트롤러**: 결혼 신청 토글, [🥚 후계자 맞이] 버튼 제공.
*   **조상 히스토리**: 대수별 은퇴 조상의 향년, 직업/칭호, 사망 원인 타임라인 리스트.

### 1.7 가문 묘지 (`/tama/grave`)
*   **묘역 리스트**: 사망한 조상들의 묘비 오브젝트 배치. 클릭 시 팝업으로 상세 생전 정보 및 유언/추모 문구 표출.

### 1.8 알 부화 페이지 (`/tama/hatch`)
*   **단계**: 종족 선택(4대 유형) $\to$ 이름 텍스트 필드(중복 확인 포함) $\to$ 알 깨기 터치 인터랙션 연출(3회 터치 시 알 크랙 발생 및 해칭 애니메이션) $\to$ 완료.

### 1.9 알림함 (`/tama/notifications`)
*   **목록**: 수신된 전투 신청(수락/거절 선택), 결혼 신청(수락/거절 선택), 완료된 전투/결혼 소식 리스트형 알림창.

---

## 2. 디자인 및 UX 원칙

*   **스토리와 표정 중심**: 스탯 게이지 수치보다는 다마고치의 표정 변화와 텍스트 말풍선 대사 위주의 UI를 전면에 드러내어 생명체를 다루는 애착 형성에 집중합니다.
*   **이동 및 상호작용 제한 표시**: 시간당 제한 횟수가 있는 액션은 버튼 우측에 `(현재횟수/최대횟수)` 배지를 의무적으로 렌더링합니다.
