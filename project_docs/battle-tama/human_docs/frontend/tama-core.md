# 기획 명세: 다마고치 코어 (스탯, 상호작용, 부화, 사망)

## 🎯 목적
하루상자의 대표 육성 콘텐츠인 '전투 다마고치'의 핵심 시스템(스탯, 기본 상호작용, 알 부화, 캐릭터 시스템, 사망)을 정의합니다. 1~5분 단위의 가벼운 플레이를 유도하며, 강함보다는 개성과 애착 형성에 집중합니다.

## 🔄 시스템 구조(Flow)

### 1. 알 부화 및 캐릭터 생성
- **종족 선택:** 귀염상, 기괴상, 평범상, 개성상 중 택 1
- **이름 짓기:** 중복 없는 이름 설정
- **본체 결정:** MVP 기준 총 20종 본체 중 랜덤 배정
- **색상 변이:** 본체별 5종 색상 팔레트 중 랜덤 배정
- **모자 배정:** 처음부터 1개의 모자(총 10종 중 랜덤) 고정 장착 (변경 불가)

### 2. 스탯 시스템
- **스탯 종류:** 😊 행복도(0~100), 🍪 배고픔(0~100), 🛁 청결도(0~100), 🦁 용기(0~100)
- **시간 경과 감소:** 
  - 행복도: 1시간당 -5
  - 배고픔: 1시간당 -4
  - 청결도: 1시간당 -4
  - 용기는 시간에 따라 감소하지 않는 성향 스탯
- **접속 시 스탯 계산:** 마지막 접속 시간 기준으로 스탯 자동 계산 적용

### 3. 기본 상호작용 (돌보기)
- **쓰다듬기:** 1시간에 10번 가능 / 100% 성공 / 행복도 +2
- **박수치기:** 1시간에 2번 가능 / 30% 확률로 성공 (성공시 행복도 +15, 실패시 변화 없음)
- **책 읽어주기:** 1시간에 5번 가능 / 50% 확률로 성공 (행복도 +3, 용기 +2)
- **간식 주기:** 1시간에 1번 가능 / 100% 배고픔 +20 / 50% 행복도 +5 / 70% 용기 +1

### 4. 사망 및 환생 시스템
- **위급 상태 진입:** 행복도, 배고픔, 청결도 모두 0 달성 시
- **유예 기간:** 72시간 내 돌보기 시 정상 복귀
- **사망 처리:** 위급 상태 72시간 지속 시 사망, 현재 육성 불가 상태가 됨
- **환생(새로운 알 부화):** 기존 다마고치와 별개의 새로운 생명(종족, 본체, 색상, 모자 랜덤) 생성

## 📱 상세 화면 UX

### 메인 홈(`/tama`) 화면 구성
- **최상단:** 현재 상태 알림("배가 고파요", "위급해요" 등)
- **중앙 (캐릭터 영역):** 다마고치 이름, 칭호, 픽셀 캐릭터, 나이, 가문명, 승리 횟수. 스탯보다 캐릭터의 상태(표정)와 대사를 가장 크게 노출.
  - *배고픔(70~100):* "오늘도 기분 좋아!"
  - *배고픔(40~69):* "심심한데?"
  - *배고픔(20~39):* "배가 조금 고픈데..."
  - *배고픔(0~19):* "간식 줘... 진짜 배고파..."
- **중단:** 현재 스탯 바 (행복도, 배고픔, 청결도, 용기)
- **하단 (액션 버튼):** 쓰다듬기, 박수, 책 읽기, 간식, 산책, 목욕, 달리기, 전투 버튼 및 남은 횟수 표시
- **최하단:** 인벤토리 (획득한 꽃 장착 등)

### 알 부화(`/tama/hatch`) 화면 구성
- 종족(4종) 선택 ➡️ 이름 입력 및 중복 확인 ➡️ 3번 터치하여 알 부화 연출 ➡️ 탄생 결과 출력

## 💾 데이터 명세

### 1. Tamagotchi Entity (기본 상태)
- `id`: Unique ID
- `userId`: 소유자 ID
- `name`: 다마고치 이름 (유니크)
- `familyId`: 소속 가문 ID
- `generation`: 세대 수 (1대, 2대 ...)
- `status`: 생존 여부 (`ALIVE`, `EMERGENCY`, `DEAD`, `RETIRED`)
- `birthDate`: 생성일 (나이 계산: 1주일 = 1살)
- `lastAccessedAt`: 마지막 접속 시간

### 2. Stats & Interactions
- `happiness`, `hunger`, `cleanliness`, `courage`: 현재 스탯 (0~100)
- `interactions`: 각 행동(쓰다듬기, 박수 등)의 최근 실행 시간 및 남은 횟수 배열

### 3. Appearance (외형)
- `species`: 종족 (귀염/기괴/평범/개성)
- `bodyType`: 본체 ID
- `colorPalette`: 색상 ID
- `hat`: 고정 모자 ID
- `flower`: 현재 장착 중인 꽃 ID

## 🚨 구현 제약 조건
- **스탯 상/하한:** 모든 스탯 연산 시 결과값은 `Math.max(0, Math.min(100, val))`로 처리.
- **접속 동기화:** 홈 화면 마운트 시 `lastAccessedAt` 기준으로 오프라인 동안 감소할 스탯을 계산하여 즉시 반영해야 함.
- **레이어 렌더링:** 픽셀 캐릭터는 "본체 ➡️ 표정(상태에 따라 자동 변화) ➡️ 장착된 꽃 ➡️ 모자" 순의 레이어 합성을 통해 렌더링.
- **사망 로직:** 서버 스케줄러(혹은 접속 시점 검사)를 통해 위급 상태 진입 시점을 기록하고 72시간 경과 여부를 판별.
