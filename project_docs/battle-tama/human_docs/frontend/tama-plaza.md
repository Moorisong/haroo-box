# 기획 명세: 다마고치 광장, 소통 및 랭킹 시스템 (`/tama/plaza`, `/tama/rank`)

## 🎯 목적
실시간 동기화의 부담을 줄이면서도 다른 유저의 살아있는 다마고치와 가볍게 교류할 수 있는 비동기 소셜 공간(광장)을 제공합니다. 이름 검색을 통한 타겟팅 교류, 하루의 일상을 공유하는 '오늘의 외치기', 그리고 동기 부여를 위한 랭킹 시스템을 정의합니다.

## 🔄 시스템 구조(Flow)

### 1. 🏡 다마고치 광장 목록 매칭
- **동작 방식:** 실시간 공간이 아니며, 광장 진입 시 서버에서 조건에 맞는 다마고치 10마리를 조회해 리스트로 반환.
- **노출 우선순위:**
  1. 현재 접속 중인 다마고치 (최대 5마리 랜덤 우선 배정)
  2. 최근 활동(7일 이내 접속) 다마고치 (우선순위 1에서 남은 슬롯 채움)
  3. 위 조건으로도 10마리가 부족할 경우 전체 다마고치 중 랜덤 조회.
- **기능:** 노출된 각 다마고치 카드를 통해 [대결 신청], [결혼 신청], [정보 보기] 액션 가능.
- **검색 기능:** 정확한 "이름 검색(Exact Match)"만을 지원하여 특정 친구나 유명 다마고치를 찾아 상호작용 가능.

### 2. 📢 오늘의 외치기 (광장 게시판)
- **개요:** 유저가 다마고치를 통해 하루 단위의 기록이나 자랑을 남기는 가벼운 소셜 게시판 기능.
- **제약 조건:** 하루에 다마고치당 1개의 글만 작성 가능. 좋아요 및 댓글 작성 지원.
- **초기화 정책:** 외치기는 순간의 소통 공간이므로 매일 자정(00:00:00)에 그날 작성된 모든 게시글, 댓글, 좋아요 데이터가 영구 삭제(초기화)됨.

### 3. 🏆 랭킹 시스템
- **분류:** 
  - *최강 전사 랭킹:* 전투 승리 횟수순
  - *겁쟁이 랭킹:* 전투 중 도망 횟수순
  - *행복왕 랭킹:* 행복도 100 달성 횟수순 (99 이하에서 100이 되는 순간만 +1 카운트)
  - *칭호왕 랭킹:* 획득 칭호 개수순
- **처리 기준:** 동률일 경우 공동 순위로 처리하며, 랭킹 뷰는 성능 최적화를 위해 실시간보다는 주기적 갱신(캐싱)을 권장함.

### 4. 🔔 알림함 (`/tama/notifications`)
- 대결 신청, 수락/거절, 대결 결과, 결혼 신청 등의 이벤트를 수신하여 사용자가 놓치지 않고 대응할 수 있게 함.

## 📱 상세 화면 UX

### 광장 메인 화면 (`/tama/plaza`)
- 상단: 이름 검색 바 🔍
- 중단: 가로 스크롤 혹은 리스트 형태의 10마리 노출 섹션.
  - "🟢 지금 놀고 있는 친구" 구역과 "🌙 최근 다녀간 친구" 구역으로 시각적 분리.
  - 다마고치 외형 픽셀 아트, 대표 칭호, 이름 노출.
- 하단/플로팅 탭: [오늘의 외치기] 메뉴 진입 버튼.

### 오늘의 외치기 모달/페이지
- 타임라인 형태의 게시글 피드.
- 각 글마다 좋아요(❤️) 수와 댓글(💬) 수 표시. "글 남기기" 플로팅 액션 버튼(FAB) 배치.

### 랭킹 페이지 (`/tama/rank`)
- 최상위 탭 UI: [최강 전사] / [겁쟁이] / [행복왕] / [칭호왕]
- 리스트 UI: 순위 메달 아이콘(1, 2, 3위 구분), 다마고치, 가문명, 랭킹 기준 수치 노출.

## 💾 데이터 명세

### 1. Shout (오늘의 외치기)
- `id`, `authorId`, `content`, `likesCount`
- `createdAt`
- (매일 자정 삭제 대상 테이블)

### 2. Notification
- `id`, `userId`, `type`: 'BATTLE_REQ', 'BATTLE_RES', 'MARRIAGE_REQ', 'SYS_MSG'
- `relatedEntityId` (전투 ID 등)
- `isRead`: boolean

## 🚨 구현 제약 조건
- 오늘의 외치기 데이터는 매일 자정에 삭제되어야 하므로 DB 스케줄러를 통한 Table Truncate 혹은 TTL(Time To Live) 인덱스를 설정하여 자동 만료 처리.
- 광장 매칭 조회 쿼리는 성능 부하를 일으킬 수 있으므로, 현재 접속자 목록(Redis 등)과 최근 접속자 캐싱 로직을 도입해 부하를 분산할 것.
