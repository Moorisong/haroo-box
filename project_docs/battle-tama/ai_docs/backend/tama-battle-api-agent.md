# tama-battle-api Agent Reference

## 📝 1. 연동 기획 명세 (tama-battle.md)
비동기 방식의 PvP 전투 매칭, 스킬 보정, 전투력 기반 승률 산정, 도망가기 확률 계산 및 승패 보상 트랜잭션을 처리합니다.

## 🤖 2. AI 개발 지침 및 설계 구조

### 🎯 목적
클라이언트에서 결과를 조작할 수 없도록 전투력 계산 및 승패 산정 로직을 서버에서 격리하고, 10분간의 전투 락(Lock)을 관리합니다.

### 📦 패키지 및 타깃 클래스 경로 구조
- `apps/server/src/controllers/tamaBattleController.ts`
- `apps/server/src/services/tamaBattleService.ts`
- `apps/server/src/repositories/battleRepository.ts`

### 🛠️ 개발 단계 (Step-by-Step 상세 로직)
1. **대결 매칭 및 스킬 락:**
   - 대결 신청 API: `BattleRequest` 생성 및 상대에게 알림 전송.
   - 대결 수락 API: 수락과 동시에 양측의 전투력 `행복도 + 배고픔 + 청결도 + (용기 × 1.5)` 계산. 스킬 발동 시(30% 확률) 용기 +10 보정 후 최종 전투력 확정. 상태를 `IN_PROGRESS`로 변경.
2. **도망가기 및 결과 산정 로직:**
   - 조건: 양측 전투력 차이가 80 이상이고 용기가 20 이하인 개체가 존재 시 30% 확률로 도망 판정. 도망 성공 시(50%) 결과 확정.
   - 도망 실패 또는 미발생 시 전투력 차이(0~30, 31~80, 81 이상)에 따른 확률(50%, 70/30%, 90/10%) 가중치 룰렛 돌려서 승자 확정.
3. **전투 락 및 보상 지급:**
   - 클라이언트에서 10분 내에 결과 요청 시 '진행 중(IN_PROGRESS)' 반환.
   - `acceptedAt` 기준 10분 경과 후 결과 요청 시 승리(행복+10, 용기+15) / 패배(행복-5, 용기-5) 및 공통(배고픔-10, 청결-10) 스탯 업데이트 트랜잭션 실행.

## 🚨 3. 철벽 코드 컨벤션 및 제약 조건
- **[300줄 분리 규칙]:** 단일 파일 300줄 하드 리밋. 승률 산정 계산식이나 확률 로직 모듈 분리.
- **[플랫폼 락]:** Node.js/Express (또는 Next.js API Routes) 서버 환경.
- **[하드코딩 금지]:** DB 트랜잭션 시 경합 조건(Race Condition)을 방지하기 위해 반드시 원자적 처리(Atomic Transaction) 사용.
